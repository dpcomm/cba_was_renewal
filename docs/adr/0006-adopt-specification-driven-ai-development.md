# ADR 0006: 스펙 주도 AI 개발(SDD) 워크플로우 도입

## Status
**Accepted**

## Date
- **2026-04-30**

## Context (배경)
- CBA WAS Renewal 프로젝트의 복잡도가 증가함에 따라, 개발자 개인의 기억력이나 파편화된 기획에 의존하여 코드를 작성하는 방식의 한계가 발생했습니다.
- AI 코딩 어시스턴트(Antigravity 등)를 적극 도입하려 하나, 프롬프트가 모호하거나 맥락이 부족하면 엉뚱한 코드를 생성하거나 기존 구조를 망가뜨리는 문제가 있었습니다.
- 이를 해결하기 위해, 인공지능이 "진실의 원천(Single Source of Truth)"으로 삼을 수 있는 명확한 명세서 중심의 개발 체계가 필요해졌습니다.

## Decision (결정)
- **문서화된 명세(Specification) 우선**: 모든 백엔드 API 설계, 비즈니스 규칙 추가/변경은 코딩 이전에 `docs/specs` 내의 마크다운 문서로 먼저 정의되어야 합니다.
- **에이전트 페르소나 헌법 제정**: `docs/agents/` 디렉토리를 신설하여 AI 에이전트의 행동 강령(SOP)을 명문화합니다.
- 코드를 생성하거나 리팩토링할 때, 개발자는 반드시 AI에게 해당 명세서와 헌법 문서를 컨텍스트로 제공하고 시작해야 합니다.

## Consequences (결과)
### 장점 (Pros)
- 인공지능이 컨텍스트(도메인 지식)를 오해하거나 환각(Hallucination)을 일으키는 빈도를 획기적으로 줄일 수 있습니다.
- "기획서 = 명세서 = 코드 = 테스트"의 라이프사이클이 일치하게 되어 프로젝트 유지보수성이 비약적으로 향상됩니다.
- 새로운 개발자가 합류하더라도 명세서와 AI의 도움을 받아 빠른 온보딩이 가능합니다.

### 단점 (Cons)
- 코드를 짜기 전에 문서를 작성하고 현행화해야 하므로, 아주 사소한 작업에서는 오버헤드로 느껴질 수 있습니다.
